# js 堆、栈、执行机制、事件循环、宏任务、微任务

# 1、什么事堆内存和栈内存

js中的内存分为堆内存和栈内存；基本数据类型存放在栈内存中，比如Number String Undefined Boolean Symbol; 像引用数据类型比如Object、Array都是存放在堆内存中，但是引用数据类型的地址指针存放在栈内存中，当我们想要用引用类型数据时，实际是从栈内存中先拿到该数据在堆内存中的地址，然后再从堆内存中取出数据。

var obj1 = {
  name: "小明"
}
var obj2 = obj1
obj2.name = "小白"
console.log(obj1.name)   // 小白

上面的代码 obj2.name 的值改变导致 obj1.name 也改变，就是因为 obj1 是引用类型的对象，obj1 地址指针和 obj2 的地址指针都是一样的。指向的都是堆内存中同一个地址的值。

# 2、js执行机制

js的最大特点就是单线程，前一个任务执行完，下一个任务才会执行，如果前一个任务很耗时就会造成阻塞。但是不代表js的代码就是一行一行执行的。 js主线程中会有一个执行栈来执行所有的任务

  • 同步任务和异步任务

js中的任务可以分为两种：同步任务（synchronous）和异步任务（asynchronous）；同步任务就是主线程上排队执行的任务，只有前一个任务执行完毕才会执行下一个任务，异步任务会进入任务队列中。

异步任务： setTimeout   ajax请求    Promise   回调

  • 事件循环

当主线程空闲之后会从任务队列中读取异步任务去执行，每当执行完再去任务队列中读取异步任务执行，这个过程会一直循环，这种运行机制就是事件循环 

console.log(1)
var timer1 = setTimeout(() => {
  console.log('timeout_1')
}, 1000)
var timer2 = setTimeout(() => {
  console.log('timeout_2')
}, 1000)
console.log(2)

这段代码的执行结果是： 1 2 timeout_1 timeout_2

  1. console.log(1) 为同步代码进入主线程执行栈执行
  2. setTimeout 为异步任务，加入任务队列， 接下来的setTimeout同样进入任务队列
  3. consoe.log(2) 为同步代码进入执行栈执行 当1.3步的同步任务执行完输出 1 2 ；此时执行栈为空，从任务队列依次取出任务执行，任务队列遵循先进先出原则，所以输出 timeout_1 timeout_2
  • 宏任务和微任务

宏任务： 一般是指script代码、setTimeout、setInterval等

微任务：es6原生的Promise中then的回调、MutationObserver、process.nextTick(callback)（node.js）

一开始学习的时候，会把同步任务、异步任务、宏任务和微任务误以为平行的四类，其实并不是。个人认为只是两种分类方法而已。

在事件循环中先执行宏任务，宏任务执行完之后再查看有没有可执行的微任务，如果有依次执行完所以的微任务（依旧是先进先出先执行的原则）；结束之后再执行下一个宏任务

看一个例子：

console.log(1)
var timer1 = setTimeout(() => {
  console.log('timeout_1')
  new Promise((resolve) => {
    console.log('promise3')
    resolve()
  }).then(() => {
    console.log('promise3_callback')
  })
}, 0)
new Promise((resolve) => {
  console.log('promise1')
  resolve()
}).then(() => {
  console.log('promise1_callback')
})
console.log(2)
new Promise((resolve) => {
  console.log('promise2')
  resolve()
}).then(() => {
  console.log('promise2_callback')
})
console.log(3)

在上面的例子中setTimeout(fn,0),0秒后执行并不会立即执行，而是等执行栈中同步任务全部执行完，主线程空闲的时候才会执行。

  1. 上面那段代码作为宏任务进入主线程执行栈执行， 输出1
  2. setTimeout 异步任务并且是宏任务，放入宏任务的队列中
  3. new Promise 执行主代码块，then后的回调放入微任务队列， 输出 promise1
  4. console.log(2) 同步代码执行 输出 2
  5. new Promise 执行主代码块，then后的回调放入微任务队列， 输出 promise2
  6. console.log(3) 同步代码执行 输出 3

执行到此时执行栈已经空了，宏代码执行完，要去任务队列中拿出任务执行，先执行所有的微任务

  1. promise1的微任务执行 输出 promise1_callback
  2. promise2的微任务执行 输出 promise2_callback

此时微任务队列已经全部执行完了，开始执行下一个宏任务

  1. setTimeout 执行，发现console.log('timeout_1') 执行并输出 timeout_1
  2. new Promise 执行主代码块，then后的回调放入微任务队列， 输出 promise3

此时宏任务执行完了，微任务队列还有任务

  1. promise3的微任务执行 输出 promise3_callback

所以最终的结果是： 1, promise1, 2, promise2, 3, promise1_callback, promise2_callback, timeout_1, promise3, promise3_callback

在网上看到一道面试题：

async function async1(){
    console.log('async1 start')
    await async2()
    console.log('async1 end')
}
async function async2(){
    console.log('async2')
}
console.log('script start')
setTimeout(function(){
    console.log('setTimeout') 
},0)  
async1();
new Promise(function(resolve){
    console.log('promise1')
    resolve();
}).then(function(){
    console.log('promise2')
})
console.log('script end')

MDN上对await的描述：await 表达式会暂停当前 async function 的执行，等待 Promise 处理完成。若 Promise 正常处理(fulfilled)，其回调的resolve函数参数作为 await 表达式的值，继续执行 async function。

若 Promise 处理异常(rejected)，await 表达式会把 Promise 的异常原因抛出。

另外，如果 await 操作符后的表达式的值不是一个 Promise，则返回该值本身。

await 后面不管是Promise还是表达式，等到await后面的语句执行完就会先返回，后面的语句相当于一个回调，放入微任务队列。

所以上面代码的执行顺序是：

  1. script代码段作为一个宏任务先执行，遇到console.log('script start'), 打印 script start

  2. setTimeout 放进宏任务队列

  3. 执行async1() 打印出 async1 start；遇到await 接着执行async2()主体部分 打印出 async2；此时async2()返回一个promise；暂停async1 函数的执行， await 让出主线程，await 返回的Promise以及 await 后的部分都要放到微任务队列

  4. 遇到 new Promise 立即执行，打印出 promise1, then后的回调放入微任务队列

  5. 遇到console.log('script end') 打印出script end

  6. 此时宏任务执行完，开始执行微任务队列的事件， 依次打印出 async1 end promise2

  7. 开始第二次事件循环，执行宏任务 setTimeOut 打印出 setTimeout

所以最终的结果是：script start, async1 start, async2, promise1, script end, async1 end, promise2, setTimeout

将上面的例子改一下

async function async1(){
  console.log('async1 start')
  let a = await new Promise(resolve => {resolve(3)})
	console.log('async1 end')
	return a
}
console.log('script start')
setTimeout(function(){
    console.log('setTimeout') 
},0)  
async1().then(res => {console.log(res)});
new Promise(function(resolve){
    console.log('promise1')
    resolve();
}).then(function(){
    console.log('promise2')
})
console.log('script end')

执行结果是：script start, async1 start, promise1, script end, async1 end, promise2, 3, setTimeout

以下是个人理解：

首次宏任务执行完的时候，微任务队列应该是console.log('async1 end') return a, Promise的回调函数then, 微任务在执行console.log('async1 end') return a的时候，执行到return后才async1才会返回一个promise回调， 这个时候才可以用then方法， 所以此时才会将async1产生的then的回调放入微任务的队尾，所以Promise的回调函数then执行完才执行async1返回的回调。
